# Je1ghtxyuN 个人网站 - 项目结构说明

## 项目概述

这是一个现代化的个人作品集网站，采用双版本架构：
1. **现代版本**：基于 React + Vite 的单页应用 (SPA)
2. **遗留版本**：传统的 HTML/CSS/JavaScript 静态网站

## 技术栈

### 核心框架
- **React 19.2.0** - 前端框架
- **Vite 7.2.4** - 构建工具
- **Firebase 12.6.0** - 后端云服务

### 主要依赖
- **lucide-react** - 图标库
- **react-markdown** - Markdown 渲染
- **remark-gfm** - GitHub Flavored Markdown 支持

### 开发工具
- **ESLint** - 代码检查
- **TypeScript 类型定义** - 类型安全
- **gh-pages** - GitHub Pages 部署

## 核心文件说明

### 配置文件
1. **package.json** - 项目配置、依赖管理和脚本命令
2. **vite.config.js** - Vite 构建配置，包含代码分割和优化设置
3. **eslint.config.js** - ESLint 代码检查配置
4. **.gitignore** - Git 忽略文件配置

### 应用入口
1. **index.html** - 主 HTML 文件
2. **src/main.jsx** - React 应用入口，包含错误边界和上下文提供者
3. **src/App.jsx** - 主应用组件，包含所有页面逻辑

### 核心组件
1. **AppContext.jsx** - 全局状态管理，包含主题、语言等状态
2. **SettingsPanel.jsx** - 设置面板组件，支持语言和主题切换
3. **ErrorBoundary.jsx** - 错误边界组件，捕获和处理运行时错误

### 工具函数
1. **dateFormatter.js** - 日期格式化工具，支持多语言日期显示
2. **dataService.js** - 数据服务，封装数据操作逻辑
3. **firebase.js** - Firebase 配置和初始化

## 开发方式

### 环境准备
```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build

# 代码检查
npm run lint

# 预览构建结果
npm run preview
```

### 开发流程
1. **启动开发服务器**：`npm run dev`，访问 http://localhost:5173
2. **代码编写**：在 `src/` 目录下编写 React 组件
3. **代码检查**：使用 `npm run lint` 检查代码规范
4. **构建测试**：使用 `npm run build` 和 `npm run preview` 测试生产构建

### 功能开发指南

#### 添加新页面
1. 在 `src/App.jsx` 中添加新的路由或页面组件
2. 更新导航菜单
3. 添加对应的国际化文本

#### 添加新组件
1. 在 `src/components/` 目录下创建新的组件文件
2. 遵循现有组件的命名和结构规范
3. 在需要的地方导入和使用

#### 添加国际化文本
1. 在 `src/locales/` 目录下的对应语言文件中添加文本
2. 使用 `useApp()` 钩子获取翻译函数
3. 通过 `t('key.path')` 使用翻译文本

## 部署方式

### GitHub Pages 部署
```bash
# 构建项目
npm run build

# 部署到 GitHub Pages
npm run deploy
```

### 部署配置
1. **vite.config.js** 中的 `base` 配置为 `/`
2. **package.json** 中的 `deploy` 脚本使用 `gh-pages`
3. 构建结果输出到 `dist/` 目录

### 部署流程
1. 确保代码已提交到 GitHub 仓库
2. 运行 `npm run deploy` 命令
3. 访问 https://je1ghtxyun.github.io 查看部署结果

## 项目架构特点

### 双版本架构
1. **现代版本**：React SPA，功能丰富，交互性强
2. **遗留版本**：静态网站，兼容性好，加载速度快

### 状态管理
1. **Context API**：全局状态管理（主题、语言等）
2. **本地状态**：组件内部状态使用 React Hooks

### 样式系统
1. **CSS 变量**：主题颜色和样式变量
2. **玻璃态设计**：使用 backdrop-filter 实现毛玻璃效果
3. **响应式布局**：支持桌面端和移动端

### 国际化
1. **多语言支持**：中文、英文、日文
2. **JSON 文件管理**：分离的语言文件
3. **动态切换**：运行时语言切换

### 性能优化
1. **代码分割**：按需加载组件
2. **图片优化**：合理使用图片格式和尺寸
3. **懒加载**：非关键资源延迟加载

## 维护指南

### 代码规范
1. 使用 ESLint 进行代码检查
2. 遵循 React Hooks 使用规范
3. 组件命名使用 PascalCase，文件使用 .jsx 扩展名

1. 使用 ErrorBoundary 捕获渲染错误
2. 添加用户友好的错误提示

## 常见问题

### 开发问题
1. **端口占用**：修改 vite.config.js 中的端口配置
2. **依赖冲突**：删除 node_modules 和 package-lock.json 后重新安装
3. **构建失败**：检查控制台错误信息，确保所有依赖正确安装

### 部署问题
1. **404 错误**：检查 vite.config.js 中的 base 配置
2. **资源加载失败**：确保 public 目录中的资源正确引用
3. **Firebase 错误**：检查 Firebase 配置和环境变量

### 性能问题
1. **加载缓慢**：检查图片大小和格式，启用代码分割
2. **内存泄漏**：确保正确清理事件监听器和定时器
3. **渲染卡顿**：优化组件渲染，使用 React.memo 和 useMemo
